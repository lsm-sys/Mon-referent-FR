# PLAN.md — реализация кнопок генерации контента

Основание: [project.md](./project.md)

## Цель

После ввода URL франкоязычной статьи и нажатия одной из четырёх кнопок приложение должно:

1. распарсить статью;
2. передать текст в AI через OpenRouter;
3. сгенерировать контент нужного формата;
4. показать результат в блоке «Resultat».

## Порядок действий

### Этап 1. Общая инфраструктура генерации

1. **Создать `lib/generateContent.ts`** — единая точка вызова AI для всех кнопок:
   - принимает `ParsedArticle` и `ActionType`;
   - обрезает длинный текст (как в `translateArticle.ts`);
   - выбирает system-prompt по типу действия;
   - возвращает готовый текст для блока «Resultat».

2. **Описать промпты для каждого действия** в отдельном файле `lib/prompts.ts`:
   - общие правила (язык, тон, запрет на выдуманные факты);
   - отдельный system-prompt для каждой кнопки (см. этапы 2–5).

3. **Обновить `/api/generate`**:
   - после `parseArticleFromUrl` вызывать `generateContent(article, action)`;
   - записывать AI-ответ в поле `result`;
   - сохранить обработку ошибок (нет контента, ошибка OpenRouter, таймаут).

4. **Обновить UI** (`ArticleForm.tsx`):
   - разные сообщения загрузки для каждой кнопки;
   - при необходимости — форматирование результата (markdown / plain text).

---

### Этап 2. Кнопка «Posts pour X»

**Формат результата:** 1–3 коротких поста для X (Twitter), до 280 символов каждый.

**Действия:**

1. Добавить промпт в `lib/prompts.ts`:
   - язык: французский;
   - стиль: лаконичный, с хуком в первой строке;
   - лимит: ≤ 280 символов на пост;
   - структура ответа: нумерованный список постов + хештеги (2–4 шт.).

2. Реализовать `generatePostsForX(article)` в `lib/generateContent.ts`.

3. Подключить в `/api/generate` для `action === "x"`.

4. Проверить на реальной франкоязычной статье: длина, читаемость, хештеги.

---

### Этап 3. Кнопка «Posts pour Instagram»

**Формат результата:** 1–2 поста для Instagram с подписью и эмодзи.

**Действия:**

1. Добавить промпт:
   - язык: французский;
   - длина подписи: 150–300 слов;
   - CTA в конце («Lien en bio», «Swipe →» и т.п.);
   - блок хештегов (10–15 шт.) отдельной строкой.

2. Реализовать `generatePostsForInstagram(article)`.

3. Подключить для `action === "instagram-posts"`.

4. Проверить: подпись не обрезается критично, хештеги релевантны теме.

---

### Этап 4. Кнопка «Infographies pour Instagram»

**Формат результата:** структура инфографики — заголовки слайдов, ключевые тезисы, идеи для визуала.

**Действия:**

1. Добавить промпт:
   - язык: французский;
   - формат: 5–7 слайдов;
   - для каждого слайда: заголовок (≤ 8 слов), 2–3 bullet points, идея иллюстрации;
   - финальный слайд: вывод / CTA.

2. Реализовать `generateInfographicsForInstagram(article)`.

3. Подключить для `action === "instagram-infographics"`.

4. Проверить: слайды логично раскрывают статью, текст помещается на карточку.

---

### Этап 5. Кнопка «Post pour Telegram»

**Формат результата:** один развёрнутый пост для Telegram-канала.

**Действия:**

1. Добавить промпт:
   - язык: французский;
   - длина: 400–800 слов;
   - структура: цепляющий заголовок, лид, основная часть, вывод;
   - допустима лёгкая markdown-разметка (`**жирный**`, списки);
   - ссылка на источник в конце.

2. Реализовать `generatePostForTelegram(article)`.

3. Подключить для `action === "telegram"`.

4. Проверить: пост читается целиком в Telegram, ссылка на статью присутствует.

---

### Этап 6. Доработка и деплой

1. **Переменные окружения на Vercel:**
   - `OPENROUTER_API_KEY`
   - `OPENAI_BASE_URL=https://openrouter.ai/api/v1`

2. **Обработка длинных статей:**
   - если текст > 12 000 символов — обрезка или суммаризация перед генерацией;
   - предупреждение в UI при обрезке.

3. **Логирование ошибок** в API (без вывода ключа в ответ клиенту).

4. **Ручное тестирование** всех 4 кнопок на 2–3 разных франкоязычных сайтах.

5. **Деплой** и финальная проверка на production.
